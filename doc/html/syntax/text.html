MMT includes a human input-oriented text syntax, which is used in particular in the jEdit editing workflow.

Structural Delimiters

The only unusual aspect of the syntax is the use of the characters ASCII 28-31 as structural delimiters. These are usually considered whitespace, and most fonts do not have glyphs for them. Therefore, typing and displaying them depends on the editor and font used.

The advantage of these delimiters is that they do not clash with any other delimiters used by notations, lexing rules, or external formats that are integrated with MMT. Therefore, they permit near-perfect error recovery and make it easy to integrate different sublanguages (e.g., for comments or informal content).

Every delimiter marks the end of a declaration or a component of a declaration.
DelimiterName in ASCIIMarks end ofComment
ASCII 28FS: file separatordocument redundant if end-of-file follows
ASCII 29GS: group separatormodule any declaration that has nested declarations, i.e., theory, view, etc.
ASCII 30RS: record separatordeclaration in particular includes and constants
redundant if end of module follows
ASCII 31US: unit separatorcomponent of declaration in particular type, definiens, notation of a constant
redundant if end of declaration follows
Note that these form a hierarchy of nested levels and that we can add SPACE as 5th level delimiter, which marks the end of a keyword.

In jEdit

The jEdit plugin provides abbreviation definitions for the delimiters. But users can define custom abbreviations by editing the abbrev file (in the settings directory), whose syntax is self-explanatory.
The Character Map plugin can help as well.

Font Support

GNU unifont is an example of a font that has reasonable glyphs for the delimiters.
It also provides many useful glyphs for mathematical operators.

Keywords

The beginning of a (component of a) declaration is marked by a keyword. Keywords for declarations are usually alphabetical words whereas keywords for declaration components are usually a single non-alphanumeric character.
