Generic Parse-Analyze-Compute-Present


The MMT parser for objects supports precedences, variable binding, and sequence arguments. Here, e.g. {} is used for Pi-binding, set comprehension, and set replacement.
Analysis infers omitted types (here: overlined) and implicit arguments (here: underlined).
Computation simplifies the term where possible (here: none).
Presentation renders terms as presentation MathML (here: displayed in Firefox).
All 4 algorithms are language-independent and fully customizable by parsing, typing, equality, and presentation rules.

Integration with other Tools

Because MMT is application-independent, it behaves very well when using MMT functionality from other systems and programming languages.

JavaScript

Because the MMT exposes many of its API functions in the QMT query language (which is itself exposed via an HTTP interface), it is straightforward to call MMT algorithms remotely.
Here is an example of how to use MMT to convert a string into a presentation MathML object.
As an extra, we use simplification in between parsing and presenting: 
   var query = qmt.present(qmt.simplify(qmt.parse(qmt.literalString(INPUTSTRING))))
   qmt.exec(query,
      function(mathml){
         // this function is run as soon as the Ajax query to the QMT server has returned
      }
   )
This uses the qmt object, which is a part of the MMT web server.

Joint Navigation

MMT provides an event listener infrastructure, which permits applications to react to content and focus changes.
IDE, theory graph, and web browser are connected so that they can control each other's focus.
For example, users can click on a node in the theory graph to navigate to its web page and then click in the web page to navigate to its source in the IDE.
