info.kwarc.mmt.api.frontend

Action

sealed abstract class Action extends AnyRef

Objects of type Action represent commands that can be executed by a Controller. In particular, these are the commands available in the MMT shell. The semantics of Actions is implemented in Controller.handle, which must contain one case for every Action.

Each subclass documents its concrete syntax and its semantics (relative to an instance "controller" of Controller). The syntax uses the following basic data types: FILE a /-separated file name, relative to controller.home URI a URI, relative to controller.base INT an integer STRING a non-empty string without whitespace CLASS a string representing a Java-style URI of a class, e.g., info.kwarc.mmt.api.frontend.Action [TYPE] for optional arguments TYPE* for whitespace-separated lists of arguments If such a type is listed as id:TYPE, then id is the name of the constructor argument of the Action, into which the concrete argument is parsed.
